SkillShipping

Adaptive learning mobile app that uses AI to create personalized, dynamic learning experiences that adapt in real-time to each learner's unique profile.

Built with React Native and Expo, SkillShipping transforms traditional static courses into intelligent, evolving learning journeys. Features include adaptive content generation, comprehensive file processing for project grading with GPT Vision, real-time learning analytics, and a Learning DNA system that creates unique learning identities.

My Role

As the sole designer and developer, I was responsible for the complete product vision, design, and implementation. My focus was on creating truly adaptive learning that feels personalized without being obvious about the algorithms behind it.

  • Designed adaptive learning system with invisible intelligence—users feel personalized content without seeing the algorithm
  • Created enhanced reading format with rhythm, pacing, ASCII diagrams, and micro-hooks that increased content engagement
  • Built intelligent project grading with multi-format support (PDF, DOCX, images), GPT Vision integration, and robust score extraction
  • Implemented hybrid adaptive system with real-time learning analysis and gap-focused content
  • Built multi-layer file reading with ~90% success rate across formats

The Challenge

Making Learning Truly Adaptive

Most "adaptive" learning platforms are just branching quizzes—if you get a question wrong, you see more questions on that topic. True adaptation means understanding how each person learns best and continuously adjusting content, pacing, and difficulty in real-time.

The challenge was to create a system that:

  • Adapts content based on learning patterns, not just quiz scores
  • Feels personalized without being creepy or obvious
  • Generates engaging content that doesn't feel AI-written
  • Grades real projects (PDFs, documents, images) with meaningful feedback
  • Works across different learning styles and goals
  • Maintains engagement over weeks or months of learning

The Approach

Invisible Intelligence

The key insight was that the best adaptive systems are invisible. Users shouldn't think "this AI is adapting to me"—they should just feel like the content is perfectly suited to their needs.

User Flows

  • Onboarding: Welcome → Skill Input → Level Selection → Duration Preference → Goal Setting → Content Type Preferences → Course Generation (30-60s)
  • Learning Loop: Course Screen → 'Up Next' Ribbon → Lecture Screen (Reading/Quiz/Project) → Completion & Feedback → Next Content Generation (Adaptive) → Loop continues
  • Project Grading: Project Screen → Upload Files (PDF/DOCX/Images) → File Preview → Grade Project → Processing → Results (Pass/Fail, Score, Feedback) → Re-grade if needed

Design Decisions

  • Invisible intelligence: Adaptive system works behind scenes—users experience content that 'just feels right' without seeing complex algorithms
  • Progressive disclosure: Single-page onboarding with visual progress, advanced features (Learning DNA, 3D Map) unlock after engagement
  • Enhanced reading format: Short paragraphs, strategic whitespace, ASCII diagrams, micro-hooks, conversational tone—feels like discovery, not homework
  • Real project grading: Native file pickers, GPT Vision for images, detailed feedback showing what GPT 'saw', re-grade option reduces anxiety

The Framework

ComprehensiveLearningTracker

The heart of SkillShipping is the ComprehensiveLearningTracker, which analyzes every interaction to build a complete picture of how each user learns:

  • Tracks quiz performance, reading time, content preferences, and struggle points
  • Identifies knowledge gaps and misconceptions
  • Builds personalized GPT prompts for next content generation
  • Validates that generated content aligns with learning goals
  • Adjusts difficulty and pacing based on progress patterns

Multi-Layer File Reading

To grade real projects, I built a robust file reading system with multiple fallback layers:

  • Layer 1: Direct text extraction for simple formats
  • Layer 2: PDFLib for binary PDFs and complex documents
  • Layer 3: Smart filename analysis when content can't be extracted
  • GPT Vision: Base64 image encoding for visual content analysis

This achieves ~90% success rate across all file types.

Technical Architecture

  • Frontend: React Native 0.81.4, Expo ~54.0.12, TypeScript 5.9.2
  • AI Services: OpenAI GPT-4, GPT-4o (Vision), Whisper API
  • File Processing: expo-document-picker, expo-image-picker, expo-file-system/legacy, react-native-pdf-lib
  • Database: Expo SQLite 16.0.8 with learning_sessions, ai_interactions, quiz_attempts, project_submissions, user_progress tables

Detailed Design

Enhanced Reading Format

Generic AI-generated content feels robotic and boring. I designed an enhanced reading format that makes learning feel like discovery:

  • Rhythm and pacing: Short paragraphs (2-3 sentences), varied sentence length, strategic whitespace
  • ASCII diagrams: Simple text-based visuals that explain concepts without images
  • Micro-hooks: Questions and teasers that pull readers forward
  • Conversational tone: Second person ("you"), contractions, relatable examples
  • Progressive complexity: Start simple, build to advanced concepts

This increased content engagement by 40% in testing.

Intelligent Project Grading

One of the most challenging features was grading real project submissions. The system:

  • Accepts PDFs, DOCX files, and images
  • Extracts content using multi-layer reading system
  • Uses GPT Vision to analyze images and visual content
  • Provides detailed feedback on what was good and what needs improvement
  • Shows users exactly what GPT "saw" for transparency
  • Allows re-grading if users disagree with the assessment

Score Extraction System

GPT returns free-form text, not structured data. I built a robust score extraction system:

  • Multi-pattern regex parsing to find scores in various formats
  • 80% pass/fail threshold with clear communication
  • Comprehensive error handling for ambiguous responses
  • Fallback to manual review if confidence is low

Refinement

Making Adaptation Truly Invisible

Early testing revealed that users felt uncomfortable when they could "see" the adaptation happening. Comments like "it's creepy how it knows I'm struggling" indicated I needed to make the intelligence more subtle.

Refinements included:

  • Removing explicit "adapting to you" messaging
  • Introducing variety in content even when focusing on gaps
  • Spacing out difficulty adjustments so they feel natural
  • Framing recommendations as suggestions, not prescriptions

After these changes, users described the experience as "perfectly paced" without attributing it to AI.

The Impact

Key Achievements

  • 40% increase in content engagement with enhanced reading format
  • ~90% success rate for file reading across all formats
  • Real-time adaptive content generation based on learning patterns
  • Intelligent project grading with GPT Vision integration
  • Users describe experience as "perfectly paced" without realizing it's AI-driven

Lessons Learned

This project taught me that the best AI experiences are the ones users don't consciously notice. When adaptation is too obvious, it feels invasive. When it's invisible, it feels magical—like the content was made just for you. The key is designing systems that enhance the experience without calling attention to themselves. Also, building robust file handling with multiple fallback layers is essential for real-world applications where users will throw every possible file format at your system.
